Development of multi-vendor business logic (command processing mechanism)

Our team has developed the following functionality of the main cross-platform business logic (thin client) of the POS terminal:
- Receiving command packages from the server side (printing a check, reading card data, displaying a form, changing operational keys, etc.) - Pre-processing of commands before sending them to the platform level for execution - Returning the results of the commands to the server (entered data on the screen, error codes / successful execution of the hardware level functions, etc.)
